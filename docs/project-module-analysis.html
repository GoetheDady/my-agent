模块总览

1. Core Runtime / API 网关

90%

负责进程启动、配置、数据库初始化、Hono API、静态文件服务和 WebSocket 升级。

  • 入口清晰，`src/main.ts` 注册全部 API 和后台服务。
  • SQLite schema、迁移和静态文件安全检查已有测试或实现。

2. Agent 管理与配置

88%

负责 Agent 创建、状态、agent.json 统一配置、工具策略和 Skill 元数据。

  • 支持多 Agent、配置 patch、reset、损坏配置恢复。
  • 后续可补更细的 UI 配置编辑和配置版本审计。

3. Agent Runtime 执行引擎

84%

负责任务执行编排：构建 system prompt、调用模型、处理工具、保存结果和失败状态。

  • 45 秒超时、abort 信号、失败释放 Agent 状态都有覆盖。
  • 生产级流式恢复、模型多供应商故障切换还未充分展开。

4. Task 队列

88%

负责 queued/running/completed/failed/canceled 生命周期，同一 Agent 串行执行。

  • 优先级领取、忙碌保护、取消、运行中恢复均有测试。
  • 多进程部署需要分布式锁或数据库级 claim 强化。

5. Chat / Session 服务

82%

负责 Web 会话、消息持久化、聊天入口和 assistant.message.persisted 生命周期事件。

  • 前端先创建 session 的约束已落实。
  • 仍可补消息分页、错误重试和更完整的流式 UI 状态。

6. Channel 外部渠道

70%

负责 Web、飞书、微信等入口映射到统一 conversation/task，并处理外部回复。

  • 飞书 WebSocket、扫码绑定、审批卡、事件解析和出站发送已有完整 MVP。
  • 微信仍是 stub，占位适配器；更多渠道还未接入。

7. Tool 与审批系统

86%

负责工具注册、工具集构建、权限策略、路径白名单和审批记录。

  • 只读默认允许、写入审批、agent.json 禁写保护都有测试。
  • 可继续补工具执行超时、审计导出和更细粒度沙箱策略。

8. Memory 记忆系统

82%

负责长期记忆、工作记忆、情景记忆、向量检索、提取、去重和 Dream Worker。

  • 记忆提取已后端驱动，工具卡注入和去重链路有测试。
  • 依赖 LanceDB native binding、Embedding API 和 LLM planner，线上稳定性仍需长期观察。

9. Profile 同步

78%

负责把稳定用户认知和 Agent 自我规则同步到 agent-scoped user.md / soul.md。

  • 迁移、去重、分类、事件记录已有测试。
  • 仍需要更成熟的人类可编辑冲突解决体验。

10. Skill 系统

80%

负责 Agent 本地 SKILL.md 文件、Skill 列表、创建、启用禁用和索引。

  • 文件边界和 agent.json 元数据分离清楚。
  • 还缺少更完整的 Skill 编辑器、校验器和导入/导出流程。

11. Delegation 委派系统

72%

负责 Agent A 把任务交给 Agent B，并在子任务完成后创建父 Agent 回调任务。

  • 核心创建、禁止递归、自委派保护和回调任务已有测试。
  • 复杂协作、并发委派 UI、跨 Agent 权限边界仍是 MVP 状态。

12. Event 事件日志

85%

负责把 task/tool/memory/channel 等运行时事件写入 SQLite，并供前端查询。

  • 按 Agent、Task、Conversation 查询和 JSON payload 往返已有测试。
  • 后续可补事件归档、过滤语法和告警规则。

13. Realtime WebSocket

75%

负责浏览器实时通知，当前更多是“状态变化提醒”，前端再通过 HTTP 拉取最新数据。

  • 订阅过滤、ping、socket 管理有测试。
  • 还不是完整事件流系统，断线恢复和消息序号可继续加强。

14. Web 工程控制台

76%

React/Vite 控制台，包含 Chat、Memory、Profiles、Agents、Channels、Tools、Skills、Tasks、Events、Architecture、Settings 页面。

  • 生产构建通过，关键 store/lib 有单元测试。
  • 存在 896KB JS chunk 警告；缺少端到端交互测试和按路由拆包。

调用链路

入口

Web Chat / Feishu / API 创建用户消息。

标准化

Channel 或 Session 映射 identity、conversation、task。

执行

Task Queue 调度 Agent Runtime，模型和工具完成任务。

记录

消息、事件、审批、记忆写入 SQLite/LanceDB。

反馈

前端轮询和 WebSocket 刷新状态；飞书发送出站消息。

P2 多进程或长期运行场景下，Task/Dream 调度需要更强的锁和恢复策略。 如果进入部署阶段，引入数据库级 claim、心跳和运行中任务恢复报告。

术语说明

MVP最小可用产品，意思是核心流程能跑通，但体验、边界处理和规模化能力还没有完全打磨。
Runtime运行时系统，负责把一次输入转成任务、模型调用、工具调用、事件和最终回复。
API 网关统一 HTTP 入口，负责把不同路径转发到对应后端路由。
WebSocket浏览器和服务端之间的长连接，用于低延迟推送状态变化。
Embedding把文本转换成数字向量，用于计算两段文字在语义上是否相似。
端到端测试用真实浏览器或真实接口模拟用户完整操作流程，比单元测试更能发现集成问题。
